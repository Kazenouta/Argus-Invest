# 风控日报模板

> 基于《风控管理系统——日报篇》设计文档  
> 存储路径：`data/reports/report_schema.parquet` + 各子表  
> 版本：v1.0  2026-04-01

---

## 一、文件结构

```
data/reports/
├── report_schema.parquet          # 报表分区结构定义（含字段说明）
├── pre_market_YYYYMMDD.parquet    # 盘前报告数据（每日生成）
├── post_market_YYYYMMDD.parquet   # 盘后复盘日报（每日生成）
├── weekly_review_YYYYMMDD.parquet # 周报复盘（每周生成）
├── example_holdings.parquet       # 示例：持仓快照
├── example_prohibited.parquet     # 示例：禁止操作清单
├── example_risk_scans.parquet     # 示例：风险扫描
├── example_trade_records.parquet  # 示例：调仓记录
├── example_compliance.parquet     # 示例：合规评级
└── example_portfolio_snapshot.parquet # 示例：持仓健康度快照
```

---

## 二、邮件标题格式

```
【{风险等级}-{核心问题标签}】Argus {报告类型} {日期}
```

**风险等级**：`高风险` / `中风险` / `观察`  
**核心问题标签**：`见顶信号` / `仓位超载` / `高频交易预警` / `规则违反` / `合规` / `无风险`

**示例**：
- `【高风险-见顶信号】Argus 盘前报告 2026-04-02`
- `【中风险-仓位超载】Argus 盘前报告 2026-04-02`
- `【警告-规则违反】Argus 盘后复盘 2026-04-01`
- `【观察】Argus 盘前报告 2026-04-02`

## 三、盘前报告（Pre-Market Report）

**推送时间**：交易日 08:30 – 09:15  
**文件名**：`pre_market_YYYYMMDD.parquet`

### 3.1 大盘昨日情绪

| 字段 | 说明 | 示例值 |
|---|---|---|
| market_sentiment | 市场情绪定性 | `多头` / `空头` / `中性` |
| hs300_change_pct | 沪深300涨跌幅% | `-0.52` |
| csi500_change_pct | 中证500涨跌幅% | `-0.87` |
| chinext_change_pct | 创业板指涨跌幅% | `-1.23` |
| volume_ratio_20d | 成交量相对20日均量倍数 | `1.3` |
| northbound_flow | 北向资金净流入(亿) | `-23.5` |
| sentiment_summary | 情绪摘要（文字描述，50字以内） | `市场缩量下跌，沪深300领跌，情绪偏谨慎` |

### 3.2 昨日板块资金流向

| 字段 | 说明 | 示例值 |
|---|---|---|
| top_sectors_by_gain | 涨幅最大板块(Top3) | `["贵金属", "银行", "煤炭"]` |
| top_sectors_by_flow_in | 资金流入最多板块(Top3) | `["贵金属", "电力设备", "银行"]` |
| top_sectors_by_flow_out | 资金流出最多板块(Top3) | `["科技", "消费", "医药"]` |
| sector_flow_summary | 板块资金小结 | `主线资金流入贵金属避险，消费和科技资金持续流出` |

### 3.3 持仓股票风险扫描

> 无风险的股票不出现在报告中。

| 字段 | 说明 |
|---|---|
| stock_code | 证券代码 |
| stock_name | 证券名称 |
| position_pct | 持仓占比%（相对于总资产） |
| risk_items | 风险项列表（文本列表） |
| risk_level | 风险等级：`安全` / `关注` / `警告` / `危险` |
| trigger_rules | 触发规则ID列表 |
| trigger_explanation | 风险说明（文字描述） |
| proximity_warning | 即将触发预警空间（%），若无则填 null |

### 3.4 仓位健康度检查

| check_item | threshold | current_value | status | action_required |
|---|---|---|---|---|
| 单票仓位上限 | 20% | 22.3% | 预警 | T+1收盘前减仓至20%以下 |
| 总仓位上限 | 85% | 75.2% | 正常 | — |
| 持仓数量 | 5-8只 | 5只 | 正常 | — |
| 现金比例下限 | 15% | 24.8% | 正常 | — |

### 3.5 观察股票池建仓条件扫描

| 字段 | 说明 |
|---|---|
| stock_code | 证券代码 |
| stock_name | 证券名称 |
| 建仓条件_调整充分 | True/False（10-30交易日调整期） |
| 建仓条件_均线多头排列 | True/False（5日线上穿10日线） |
| 建仓条件_站稳均线 | True/False（站稳5日线） |
| overall_status | `未满足` / `接近满足` / `已满足` |
| current_price | 当前价 |
| ma5 | 5日均线 |
| ma10 | 10日均线 |
| days_since_adjustment | 调整天数 |

### 3.6 今日禁止操作清单

| 字段 | 说明 |
|---|---|
| stock_code | 证券代码 |
| stock_name | 证券名称 |
| prohibited_action | 禁止操作（文字描述） |
| reason | 禁止原因 |
| related_rules | 关联规则ID列表 |
| alternative_suggestion | 替代建议（如有） |

## 四、盘后复盘日报（Post-Market Report）

**推送时间**：交易日 15:30 – 20:00  
**文件名**：`post_market_YYYYMMDD.parquet`

### 4.1 仓位健康度快照

| 字段 | 说明 | 示例值 |
|---|---|---|
| 持仓数量 | 持仓标的数量 | `5` |
| 现金比例_pct | 现金占比% | `24.8` |
| 单票最大占比_pct | 单票最大占比% | `22.3` |
| 总仓位_pct | 总持仓占比% | `75.2` |
| 较昨日仓位变化_pct | 较昨日仓位变化% | `-1.2` |
| 持仓明细 | 持仓明细列表（子表） | 见示例数据 |

**持仓明细子表字段**：stock_code, stock_name, position_pct, cost_basis, current_price, unrealized_pnl_pct, market

### 4.2 调仓操作记录与逻辑审查

> 用户上传调仓记录时**必须**填写调仓逻辑（文字描述）。

| 字段 | 说明 | 示例值 |
|---|---|---|
| 操作日期 | 日期 | `2026-04-01` |
| stock_code | 证券代码 | `600114` |
| stock_name | 证券名称 | `东睦股份` |
| 操作类型 | 买入/卖出/减仓/清仓 | `买入` |
| 成交价格 | 元 | `15.20` |
| 成交数量 | 股 | `100` |
| 成交金额 | 元 | `1520.0` |
| 用户填写的调仓逻辑 | 文字（用户手填） | `觉得跌了可以补一点，摊薄成本` |
| AI逻辑审查结果 | 逻辑驱动类型 | `情绪驱动` |
| 情绪标签 | 如：不甘心/怕踏空/过度自信 | `["不甘心", "试图摊薄成本"]` |
| 是否符合止盈规则 | True/False | `False` |
| 是否违背盘前禁止指令 | True/False | `True` |
| 后续操作建议 | 文字 | `注意R021，建议止损而非加仓` |
| 合规性评价 | 文字 | `违反R021禁止下跌趋势加仓规则，为情绪驱动操作` |

### 4.3 盘中思考记录

> 核心目的：强制外化盘中判断，而非仅在脑中过一遍。

| 字段 | 说明 | 示例值 |
|---|---|---|
| 思考时间 | datetime | `2026-04-01 10:23:15` |
| 关联标的 | 证券代码（可空） | `600114` |
| 思考类型 | 类型枚举 | `个股分析` |
| 思考内容 | 文字原文 | `觉得今天东睦能反弹，补了点仓` |
| 判断依据 | 文字（可空） | `技术面超跌 + 分时图有支撑` |
| AI标签摘要 | 标签列表 | `["情绪驱动", "无基本面支撑"]` |
| 是否关联操作 | True/False | `True` |
| 操作结果 | 文字（可空） | `补了100股，收盘微涨0.3%` |

### 4.4 合规评级

**评级档位（4档）**：

| 评级 | 说明 |
|---|---|
| 优良 | 完全符合规则，无违规操作 |
| 合格 | 有操作但无明显违规，或仅有轻微偏差 |
| 警告 | 有明显违规操作，但未造成较大损失 |
| 危险 | 严重违反核心规则，或出现典型错误模式 |

**评级数据字段**：

| 字段 | 说明 |
|---|---|
| 评级日期 | 日期 |
| 合规评级 | 优良/合格/警告/危险 |
| 评级说明 | 文字说明本次评级原因 |
| 今日违规次数 | 整数 |
| 违规规则列表 | 规则ID列表 |
| 情绪驱动操作次数 | 整数 |
| 逻辑驱动操作次数 | 整数 |
| 综合评价 | 文字 |

### 4.5 规则违反分析

> 每日盘后如有违规，逐条记录分析。

| 字段 | 说明 |
|---|---|
| 违反规则ID | 如 `R021` |
| 违反规则名称 | 如 `禁止下跌趋势中加仓` |
| 发生时间 | datetime |
| 标的 | 证券名称 |
| 操作描述 | 文字描述本次操作 |
| 历史镜像 | 过去何时曾犯过类似错误 |
| 错误模式定性 | 如 `情绪驱动-越跌越买` |
| 定性分析 | 文字分析 |

### 4.6 新增弱点提示

> 发现新错误模式时触发弱点更新流程。

| 字段 | 说明 |
|---|---|
| 发现时间 | datetime |
| 疑似弱点描述 | 文字描述 |
| 涉及标的 | 证券名称列表 |
| 发生次数 | 整数 |
| 证据数据 | 文字描述证据 |
| 关联现有弱点 | 弱点ID列表 |
| 用户确认状态 | `待确认` / `已确认纳入` / `已排除` |
| 建议规则 | 文字建议 |

### 4.7 周报——思考回顾机制

**生成时间**：每周最后一个交易日盘后  
**文件名**：`weekly_review_YYYYMMDD.parquet`

| 字段 | 说明 |
|---|---|
| 周报周期开始 | 日期 |
| 周报周期结束 | 日期 |
| 本周思考总数 | 整数 |
| 本周操作总数 | 整数 |
| 逻辑驱动操作数 | 整数 |
| 情绪驱动操作数 | 整数 |
| 正确的大盘判断数 | 整数 |
| 大盘判断总数 | 整数 |
| 有效大佬观点数 | 整数（跟踪下来靠谱的） |
| 本周思考摘要 | 文字 |
| 有效思考框架 | 文字列表 |
| 重复犯错模式 | 文字列表 |
| 下周观察重点 | 文字列表 |
| 下周禁止操作提醒 | 文字列表 |

## 五、合规评级算法（参考）

```
if 违规次数 == 0 and 情绪驱动操作 == 0:
    评级 = "优良"
elif 违规次数 <= 1 and 情绪驱动操作 <= 1:
    评级 = "合格"
elif 违规次数 >= 2 or 情绪驱动操作 >= 3:
    评级 = "警告"
else:
    评级 = "危险"
```
