# 私人投顾系统——知识库篇

> 本文档基于 2026-03-31 的讨论整理，是知识库模块的第一版设计文档。后续基于实际开发持续优化和迭代。

---

## 一、设计背景与定位

知识库是系统的"进攻侧"核心，负责从外部持续获取高质量信息，为策略分析师提供决策原材料。

**在系统中的位置**：
- 输入端：为机会嗅探模块（Opportunity Sniffer）提供原材料
- 数据源：喂养策略分析师进行逻辑映射和观点整合
- 权重校准：通过历史验证结果，动态调整各来源的置信度

**核心价值**：
- 打破信息茧房：跟踪用户真实关注的大V，从其历史观点与实际走势的对照中判断可信度
- 主动预警：持仓股利空早发现，大V观点变化早识别
- 逻辑外化：记录大V判断依据，供用户在事后复盘时反思自己的思考模式

**与机会嗅探模块的关系**：
机会嗅探模块的数据核心来源是用户经常关注的雪球大V。微信公众号、分析师研报作为辅助补充，提供系统性更强的分析视角。公司公告作为风控侧的补充，用于持仓股的利空监控。

## 三、雪球数据源

### 3.1 定位

雪球是机会嗅探模块的核心数据来源。

**数据来源**：用户实际关注的大V列表（非全网爬取）
- 大V的实时发言 → 市场观点的第一手来源
- 大V的历史发言 → 用于判断其分析框架和胜率
- 大V的观点变化 → 结合市场走势，评估其逻辑有效性

**为什么是雪球**：
- 大V更新频繁，观点时效性强
- 历史数据完整，可回溯验证
- 与实际走势对照后，能形成大V的置信度权重
- 覆盖多种投资风格（价值、趋势、宏观等）

**难点**：
- 需要登录态（部分内容需要关注才能查看）
- 反爬机制较严格
- 大V列表初始由用户提供，后续可补充

### 3.2 爬取策略

#### 目标数据
- **用户信息**：UID、用户名、粉丝数、关注数、简介（判断大V风格）
- **发言内容**：原创帖、评论、转发，带时间戳
- **组合信息**：模拟盘/实盘组合，持仓明细、仓位、收益曲线
- **互动数据**：点赞数、评论数（辅助判断观点受认可程度）

#### 重点关注的字段
- 发布时间和内容（用于时间线对齐）
- 标的提及（股票代码/名称）
- 观点倾向（买入/卖出/持有/观望）
- 判断依据（逻辑/宏观/技术面/基本面）

#### 爬取频率
- 大V发言列表：每30分钟轮询一次（限制频率，避免封禁）
- 组合持仓快照：每日收盘后一次
- 用户详情（粉丝数等）：每周更新一次

### 3.3 大V列表维护

**来源**：用户主动提供其关注的大V列表

- 用户在雪球上关注哪些大V，系统就跟踪哪些
- 初期由用户提供，系统负责持续抓取和更新
- 后续可扩展"大V推荐"功能：系统按投资风格分类推荐候选大V供用户添加

**大V分级**（由用户设定或AI辅助判断）：
- **核心跟踪**：用户最信任的1-3人，权重最高，每次必读
- **重点跟踪**：5-10人，历史观点与走势匹配度较高
- **辅助参考**：其余大V，用于多元视角对照，不影响核心判断

> 分级不是固定的，系统会根据历史表现动态调整各人的置信度权重。

### 3.4 逻辑映射表构建

**目的**：将大V的历史观点与后续市场走势对照，验证其逻辑有效性。

**映射方式**：
```
大V观点（时间T）→ 市场实际走势（时间T+Δ）→ 判断对/错/无关
```

**记录字段**：
- 大V名称
- 发表时间
- 观点内容（买入/卖出/持有标的X，理由Y）
- 市场后续走势
- 判断结果（成功/失败/无关）
- 关联宏观背景（如：2020年疫情、2022年加息等）

**权重更新机制**：
- 连续3次判断正确 → 该大V观点权重 +1档
- 连续2次判断错误 → 该大V观点权重 -1档
- 权重影响机会嗅探模块中该大V观点在最终输出中的占比

## 四、微信公众号数据源（补充）

### 4.1 为什么要人工下载

微信公众号的内容没有公开的批量获取接口，原生的爬虫方案需要破解微信的封闭生态，复杂度极高且容易失效。

**可行的替代方案**：
- 人工定期下载 + 上传到系统
- 优点：数据质量高，无需技术对抗
- 缺点：无法实时，依赖人工介入

### 4.2 目标来源

**新财富分析师排行榜**：
- 选择排名靠前的团队（综合排名前20，或各行业前5）
- 覆盖主要研究领域：宏观、策略、消费、医药、TMT、金融等
- 每个领域保持2-3个团队的覆盖

**补充来源（可选）**：
- 头部公募/私募基金经理的公众号
- 有影响力但不活跃于雪球的投资人

### 4.3 数据录入流程

```
人工发现文章 → 下载（PDF/截图/文字） → 上传至指定目录
                                    ↓
                          系统解析并提取关键信息
                                    ↓
                          入库至文章数据库
```

**录入内容**：
- 文章标题、作者、发布时间
- 原文内容（文字版）
- 关联标的（系统提取或人工标注）
- 文章类型（行业深度/公司跟踪/宏观策略/事件点评）

### 4.4 录入频率目标

- 核心团队文章：每周至少1-2篇
- 重要事件点评：24小时内录入
- 总量目标：每周10-20篇文章

## 五、分析师研报数据源（补充）

### 5.1 获取渠道

分析师研报可以通过以下渠道获取：

**首选：雪球机构账号**
- 许多券商研究所和头部券商已在雪球注册官方账号，发布研究报告
- 雪球上的研报发布比传统渠道更便捷，部分免费可见
- 优点：与雪球大V发言共用一套爬虫体系，无需额外开发
- 缺点：覆盖不全（只有入驻雪球的机构），时效性参差

**备选：专业研报平台（待验证）**
- 东方财富研报中心：部分免费摘要，全文付费
- 慧博投研资讯：部分免费，数据质量参差
- 同花顺 iFinD：付费订阅

> 实际优先级：雪球机构账号（如果覆盖足够）> 上述备选渠道

### 5.2 目标数据

- **报告基本信息**：标题、作者、券商、发布时间、覆盖标的
- **报告摘要**：核心观点、目标价、评级
- **报告全文**（如可获取）：详细逻辑推导、数据支撑、风险提示
- **评级分布统计**：强烈买入/买入/持有/减持/卖出（用于市场情绪辅助判断）

### 5.3 与雪球大V发言的区别

| 维度 | 雪球大V | 雪球机构账号/研报 |
|---|---|---|
| 时效性 | 实时，随时更新 | 定期发布（可能滞后） |
| 内容风格 | 观点碎片化，带情绪 | 系统性更强，逻辑完整 |
| 数据支撑 | 少 | 多（数据图表） |
| 获取难度 | 需登录态爬虫 | 同上 |

两者互补：大V提供实时情绪和灵活判断，机构研报提供系统性和深度。

## 六、公司公告数据源

### 6.1 数据来源

**首选：巨潮资讯网（www.cninfo.com.cn）**
- 官方指定上市公司信息披露网站
- 公告类型全：定期报告、重大事项、融资公告、监管问询等
- 提供公告列表和详情页 API（部分免费）

**备选：上交所/深交所官网**
- 上市公司公告披露页面
- 数据权威，但页面结构不如巨潮统一

**其他来源**：
- 东方财富股吧/公司公告频道
- 巨潮资讯的 API 接口

### 6.2 目标数据

#### 持仓/观察仓股票相关
- **重大事项公告**：并购重组、股权变动、重大合同
- **业绩公告**：年报、季报、业绩预告（业绩变脸/超预期）
- **监管问询函**：被监管机构点名，说明存在潜在问题
- **风险提示公告**：退市风险警示、其他风险警示
- **股东增减持公告**：重要股东增减持动向

#### 全市场扫描（可选）
- 定向增发、股份回购等影响资金面的公告
- 行业政策变化（通过公告中的行业影响分析判断）

### 6.3 预警机制

| 公告类型 | 严重程度 | 动作 |
|---|---|---|
| 退市风险警示（*ST） | 极高 | 立即推送，触发减仓建议 |
| 重大利空（业绩变脸、重大事故） | 高 | 当日推送，触发减仓评估 |
| 监管问询函 | 中 | 次日报推送，提示关注 |
| 重要股东大幅减持 | 中高 | 次日报推送，结合大盘判断 |
| 业绩超预期/行业政策利好 | 低 | 归档至机会池，供后续分析 |

### 6.4 财报结构化提取

对年报/季报进行结构化提取，辅助基本面分析：

- **营收/利润**：同比、环比变化
- **现金流**：经营现金流 vs 净利润匹配度
- **负债率/杠杆**：偿债能力评估
- **分业务收入**：主营构成变化
- **管理层讨论**：对未来经营方向的描述

> 注：完整的财报分析需要结合行业对比和历史趋势，知识库仅做结构化提取，深度分析由策略分析师模块负责。

## 七、数据存储设计

### 7.1 数据库表结构

```
knowledge_base/
├── xueqiu/
│   ├── xq_users.db         # 大V用户信息
│   ├── xq_posts.db        # 大V发言记录
│   ├── xq_portfolios.db   # 大V组合快照
│   └── xq_logic_map.db    # 逻辑映射表（观点 vs 走势验证）
├── wechat/
│   └── articles.db         # 微信公众号文章
├── research/
│   └── reports.db          # 分析师研报
└── announcements/
    ├── ann_list.db         # 公告索引（全市场）
    └── ann_portfolio.db    # 持仓相关公告（快速检索）
```

### 7.2 统一标注字段

所有知识库数据入库时，统一标注以下字段（便于后续检索）：

- `source`：来源（xueqiu/wechat/research/announcement）
- `author`：作者/大V名称
- `published_at`：发布时间
- `fetched_at`：入库时间
- `tickers`：关联标的（股票代码列表）
- `type`：内容类型（观点/数据/事件/分析）
- `confidence_weight`：置信度权重（1-5档，初始为3）
- `tags`：自定义标签（方便后续筛选，如"宏观""行业""公司"等）

## 九、待细化事项

以下内容在下一版设计中进一步明确：

1. **雪球爬虫实现**：登录态处理、Cookie 维护、反爬策略绕过方案
2. **雪球机构账号覆盖度**：调研已入驻雪球的券商研究所数量和发布频率，评估是否能满足研报需求
3. **微信公众号人工录入流程**：具体操作界面、文件格式、上传频率
4. **巨潮资讯 API**：接口文档、调用频率限制、数据字段
5. **大V评级权重算法**：具体的权重计算公式，连续正确/错误的判断标准
6. **财报结构化提取**：年报全文解析的技术方案（PDF 解析 + NLP 提取）
7. **知识库检索机制**：当机会嗅探模块触发时，如何快速从知识库中检索相关内容
