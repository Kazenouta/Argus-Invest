# Argus-Invest 私人投顾系统设计方案

> 本文档基于 2026-03-22 至 2026-03-31 期间的讨论整理，是系统的第一版综合设计文档。详细模块设计见各子文档。

---

## 一、项目概述

### 1.1 核心目标

构建一个基于"人机协同"模式的私人投资顾问系统，通过 AI 实现：

- **防御侧**：通过复盘用户历史交易行为发现错误模式，建立严格纪律约束，防止情绪化交易，确保本金与利润安全
- **进攻侧**：复刻并跟踪资深投资者的深度研究逻辑，从市场数据中识别投资风险和机会

### 1.2 核心价值主张

> "风控的本质不是预测市场，而是约束人性。"

系统不是要替代用户做决策，而是帮助用户在关键时刻"管住手"——用规则和历史教训约束非理性操作，用大 V 的深度逻辑扩展视野。

### 1.3 设计原则

**约束，而非预测。** 系统不追求预测市场走势，而是通过规则和历史教训，在关键时刻约束用户的非理性行为。

**外化，而非内省。** 人的思考如果只在脑子里过一遍，事后很难复盘。系统强制用户把判断依据、操作逻辑外化记录，通过定期回顾形成真正的反思闭环。

**盘前盘后下功夫。** 风控的核心机制放在盘前的预防提醒和盘后的合规复盘。

**规则从历史中来，到未来中去。** 初始规则来自历史交割单中提炼的错误模式，以及止盈篇中经过验证的止盈策略。规则不是一成不变的，而是随用户弱点更新而动态调整。

## 二、系统角色

系统内部模拟两个核心人格角色，协同工作：

### 角色 A：风控管理者（Defensive Strategist）

- **原型来源**：散户经常犯的错误，以及用户自身的历史交易教训
- **核心职责**：
  - 监控持仓风险（仓位、见顶信号、规则违反）
  - 基于弱点画像提供行为约束建议
  - 执行止盈止损纪律
  - 每日生成风控评估报告
- **性格特征**：保守、严谨、对违规操作零容忍
- **输出风格**：直接、严肃、有话直说，不给用户留借口

### 角色 B：策略分析师（Opportunity Analyst）

- **原型来源**：资深投资者的投资哲学（深度研究、逆向思维、周期共振）
- **核心职责**：
  - 关注市场行情和宏观数据
  - 每日抓取大 V 最新观点，结合知识库进行逻辑映射
  - 基于基本面和宏观逻辑发现潜在机会
  - 负责仓位管理建议
- **性格特征**：敏锐、深度、具有逆向思维。不追逐热点，只关注价值与预期的错配
- **输出风格**：分析性、启发式，引导用户独立思考

## 三、数据架构

### 3.1 三类数据源

| 数据类型 | 来源 | 存储方式 | 用途 |
|---|---|---|---|
| **用户私有数据** | 用户上传 | 本地数据库 | 风控判断、弱点分析、个性化规则 |
| **大 V 知识库** | 互联网采集 + 用户补充 | 本地数据库 | 机会嗅探、逻辑映射、观点权重校准 |
| **市场公共数据** | 同花顺/公开 API | 定时获取 | 行情计算、信号监控、报告生成 |

### 3.2 用户私有数据

#### 静态画像
- 风险偏好（保守/稳健/激进）
- 资金总量
- 投资目标（年化收益目标、最大回撤限制）

#### 动态持仓
- 实时同步：股票代码、数量、成本价、当前市值、浮盈/浮亏比例
- 每日更新：收盘后自动上传至本地数据库

#### 历史交割单
- **核心训练集**：用于训练"行为分析模块"
- 识别习惯性错误模式（越跌越买、追涨杀跌、高位加仓等）
- 上传后由系统自动分析，生成《用户弱点清单》

### 3.3 大 V 知识库

#### 数据来源（详见《私人投顾系统——知识库篇》）
- 雪球大V发言（核心）：用户关注的大V列表，实时抓取发言内容
- 雪球机构账号：新财富上榜团队的研报发布渠道
- 微信公众号：新财富分析师团队的系统性文章（人工录入）
- 公司公告：巨潮资讯爬取，持仓股利空监控

### 3.4 市场公共数据

| 数据类型 | 来源 | 更新频率 | 用途 |
|---|---|---|---|
| 个股 K 线、成交量、技术指标 | 同花顺（爬虫） | 每日收盘后 | 持仓监控、见顶信号检测 |
| 大盘指数（沪深300/中证500/创业板） | 同花顺 | 每日收盘后 | 大盘情绪判断 |
| 板块资金流向 | 同花顺（爬虫） | 每日收盘后 | 盘前报告板块扫描 |
| 宏观数据（金/铜期货、美元指数） | 公开 API | 每日 | 策略分析师宏观参考 |
| 公告资讯 | 公开渠道 | 实时 | 突发利空检测 |

> 注：同花顺数据获取通过爬虫实现，具体方案另待细化。

## 四、系统模块架构

系统由以下核心模块组成，形成"分析 → 监控 → 提醒 → 反思"的闭环：

```
┌─────────────────────────────────────────────────────────┐
│                    Argus-Invest 系统                     │
│                                                          │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐ │
│  │  行为镜像引擎  │───▶│  指标监控层   │───▶│  双报提醒系统 │ │
│  │  (弱点分析)   │    │  (规则触发)   │    │  (盘前+盘后)  │ │
│  └──────────────┘    └──────────────┘    └──────────────┘ │
│         │                   │                   │        │
│         ▼                   ▼                   ▼        │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐ │
│  │  用户弱点画像  │    │  止盈规则库   │    │  机会嗅探模块 │ │
│  │  (动态更新)   │    │  (四类策略)   │    │  (大V逻辑映射)│ │
│  └──────────────┘    └──────────────┘    └──────────────┘ │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 4.1 行为镜像引擎（Behavior Mirror Engine）

**功能**：深度分析历史交割单，构建用户"弱点画像"。

**输入**：用户过去若干年的交易记录（Excel/CSV）。

**处理流程**：
1. **场景聚类**：识别用户在什么情况下亏钱最多
   - 股价跌破60日线后补仓
   - 高位爆量时追涨
   - 单票仓位超过30%
   - 大盘下跌时不止损反而加仓
2. **错误模式提取**：生成《用户弱点清单》
   - 示例："您在'下跌趋势中补仓'的操作共15次，平均亏损12%，最大回撤35%"
3. **规则映射**：将弱点转化为具体风控规则，注入止盈规则库

**输出**：动态规则库（关联弱点标签，供后续合规评判使用）

### 4.2 指标监控层（Index Monitoring Layer）

**功能**：实时扫描市场数据，提供触发风控规则的"信号"。

**监控指标**：

| 类别 | 指标 | 阈值 | 触发动作 |
|---|---|---|---|
| 仓位健康度 | 单票仓位 | > 25% | 触发减仓警告 |
| 仓位健康度 | 总仓位 | > 90% | 触发轻仓警告 |
| 仓位健康度 | 持仓数量 | < 5 或 > 8 | 触发仓位调整提醒 |
| 仓位健康度 | 现金比例 | < 10% | 触发保留现金提醒 |
| 见顶信号 | 成交量/换手率 | ≥ 2-3倍于均量 | 触发高位警示 |
| 见顶信号 | 股价分位数 | > 前10%分位数 | 触发高位警示 |
| 见顶信号 | 技术背离 | RSI/MACD 顶背离 | 触发减仓提醒 |
| 移动止盈 | 均线价格 | 跌破20日/60日线 | 触发减仓提醒 |
| 移动止盈 | 最高点回撤 | 回撤超过15-20% | 触发止盈提醒 |
| 突发利空 | 重大公告/新闻 | 检测到重大利空 | 无条件触发减仓提醒 |

**特殊规则（基于弱点画像动态生成）**：
- 若用户历史上有"高位加仓"错误模式：对当前高位标的额外加强监控
- 若用户历史上"盈利变亏损"频率高：移动止盈线设置更紧凑

### 4.3 止盈规则库（Profit-Taking Rules）

基于《风控管理系统——止盈篇》讨论，初始规则库包含四类止盈策略：

#### 策略一：仓位管理被动止盈
- 持仓标的控制在 5-8 只
- 单票仓位不超过 25%，超过上限无条件减仓
- 保留至少 10% 现金仓位

#### 策略二：顶部指标主动止盈
- 股价进入历史高位（前10%分位数）→ 进入高度警惕
- 成交量/换手率暴增 2-3 倍以上 → 天量见天价
- RSI/MACD 顶背离 → 趋势逆转信号

#### 策略三：移动止盈
- 成本线保护法：股价上涨20%后，止盈线移动至成本价+5%
- 关键均线防守：激进版（20日线）、稳健版（60日线）
- 最高点回撤法：设定回撤比例（如15%），随股价新高自动上移

#### 策略四：突发利空无条件止盈
- 持仓股突发重大利空 → 立即减半仓，不侥幸、不等待

### 4.4 双报提醒系统（Dual Report System）

每日生成两份结构化报告，详见《风控管理系统——日报篇》：

- **盘前报告（Pre-Market）**：08:30 – 09:15
  - 大盘昨日情绪
  - 板块资金流向
  - 持仓风险扫描（含"即将触发"预警）
  - 仓位健康度检查
  - 观察股票池建仓条件扫描
  - 今日禁止操作清单

- **盘后报告（Post-Market）**：15:30 – 20:00
  - 仓位健康度快照
  - 调仓操作记录与逻辑审查
  - 盘中思考记录
  - 每日合规评级
  - 规则违反分析
  - 新增弱点提示
  - 周报——思考回顾（每周一次）

### 4.5 机会嗅探模块（Opportunity Sniffer）

**功能**：基于大 V 知识库，在市场新闻/财报发布时，自动检索类似情境下的历史判断，为用户提供"大 V 视角解读"。

**工作流程**：
1. 输入：当日市场新闻、财报或宏观事件
2. 检索：在知识库中寻找"类似情境"（如2020年疫情、2022年加息等）
3. 映射：调取大 V 在类似情境下的判断和后续市场走势
4. 输出：生成"大 V 视角解读"，判断当前是噪音还是机会

**逻辑权重机制**：
- 大 V 的历史观点被验证有效 → 适当提高该大 V 观点权重
- 大 V 的观点多次与市场走势不符 → 降低其权重
- 不同大 V 观点矛盾时 → 展示多方视角，由用户自行判断

### 4.6 用户弱点画像（Weakness Profile）

#### 初始化
- 基于用户历史交割单分析，生成初版《用户弱点清单》
- 由用户逐条确认后正式启用

#### 动态更新
- 每日复盘时检查是否存在新的错误模式
- 发现新模式 → 提示用户人工确认
- 确认纳入 → 更新弱点清单，关联对应规则

## 五、工作流设计

### 5.1 初始化阶段

```
用户上传历史交割单
        │
        ▼
系统自动分析 → 生成《用户弱点清单》（初版）
        │
        ▼
用户逐条确认弱点清单
        │
        ▼
系统生成初始规则库（结合弱点画像 + 止盈四策略）
        │
        ▼
系统初始化完成，进入日常运行
```

### 5.2 日常运行流程（交易日）

```
收盘后（15:30前）
  │
  ├── 数据同步
  │    ├── 用户上传当日持仓数据
  │    ├── 用户上传调仓记录（含操作理由）
  │    ├── 抓取同花顺大盘/板块数据（爬虫）
  │    └── 更新大V知识库最新动态
  │
  ▼
AI 双重分析（15:30 - 20:00）
  │
  ├── 风控官
  │    ├── 扫描持仓风险（见顶信号、仓位违规）
  │    ├── 对比当日操作与弱点清单
  │    ├── 生成盘后复盘日报
  │    └── 若发现新错误模式 → 触发弱点更新流程
  │
  └── 策略分析师
       ├── 扫描观察股票池建仓条件
       ├── 抓取大V最新观点
       └── 生成机会嗅探报告（可选）

  ▼
盘后复盘日报发送至用户（20:00前）

─────────────────────────────────────────────

次日 08:30 前
  │
  ├── 抓取隔夜数据（同花顺爬虫）
  ├── 生成盘前报告
  │    ├── 大盘昨日情绪
  │    ├── 板块资金流向
  │    ├── 持仓风险扫描
  │    ├── 仓位健康度检查
  │    ├── 观察股池建仓条件扫描
  │    └── 今日禁止操作清单
  │
  └── 盘前报告发送至用户（08:30）

─────────────────────────────────────────────

盘中
  │
  └── 用户随时录入盘中思考（MD文件或前端页面）
```

### 5.3 突发利空应急流程

```
触发：持仓股突发重大利空（公告/新闻）
         │
         ▼
风控官立即计算：
  - 当前持仓数量和浮盈情况
  - 若不处理，最大损失估算
         │
         ▼
策略分析师评估：
  - 是情绪错杀？还是基本面恶化？
  - 大V在类似情境下如何判断？
         │
         ▼
综合建议：
  - 情绪错杀 → "卧倒不动"或"小仓位右侧试探"
  - 基本面恶化 → "坚决减仓"
         │
         ▼
若用户执行反向操作（如加仓）：
  → 强制触发"反思三问"，回答后方可执行
  → 记录本次操作，供后续复盘分析
```

### 5.4 周报回顾流程

```
每周五/周末
         │
         ▼
汇总本周所有盘中思考记录
         │
         ▼
逐条评估：
  - 大盘判断是否正确？
  - 个股操作是否合规？
  - 大V观点是否靠谱？
         │
         ▼
生成周报：
  - 本周思考摘要
  - 正确率统计
  - 有效思考框架提炼
  - 重复犯错模式警示
  - 下周观察重点
         │
         ▼
触发弱点画像更新（如有新模式）
```

## 六、系统边界与用户决策链

```
┌──────────────────────────────────────────────┐
│                   用户                        │
│  （保留最终决策权，AI 提供建议而非指令）          │
└──────────────────┬───────────────────────────┘
                   │
         ┌────────▼────────┐
         │    AI 建议层     │
         │  （分析、预警、   │
         │   建议、提醒）    │
         └────────┬────────┘
                   │
         ┌────────▼────────┐
         │    报告生成层    │
         │  （盘前/盘后/    │
         │   周报）         │
         └────────┬────────┘
                   │
         ┌────────▼────────┐
         │    数据采集层    │
         │  （自动/定时）    │
         └─────────────────┘
```

**关键原则**：
- 系统的所有输出都是"建议"，用户保留最终决策权
- 所有操作都会被记录，供后续复盘使用
- 风控的核心手段是"盘前提醒 + 盘后复盘"，而非盘中实时拦截

## 七、技术架构概要

### 7.1 技术选型（待确认）

|---|---|---|

| 部署方式 | 本地运行（Mac/树莓派/VPS） | 数据留存在本地 |

### 7.2 数据存储方案

**统一采用 DuckDB + Parquet**

- 所有结构化数据（持仓、调仓、规则、弱点画像、大V知识库、市场行情等）均以 DuckDB + Parquet 格式存储
- Parquet 作为持久化文件格式，DuckDB 作为查询引擎
- 优势：轻量、嵌入式、无需独立数据库服务、适合本地部署、支持 SQL 查询

**目录结构**：

```
Argus-Invest/
├── data/
│   ├── user/
│   │   ├── portfolio.parquet       # 持仓数据
│   │   ├── trades.parquet          # 调仓记录
│   │   ├── weakness.parquet         # 弱点画像
│   │   └── thinking.parquet         # 盘中思考记录（前端录入）
│   ├── kv/
│   │   ├── xueqiu_posts.parquet    # 雪球发言记录
│   │   ├── xueqiu_portfolios.parquet # 大V组合快照
│   │   ├── logic_map.parquet       # 逻辑映射表
│   │   ├── wechat_articles.parquet # 微信公众号文章
│   │   └── research_reports.parquet # 分析师研报
│   ├── market/
│   │   ├── daily/                  # 每日行情快照（Parquet）
```
