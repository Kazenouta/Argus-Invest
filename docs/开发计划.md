# Argus-Invest 开发计划

> 本文档基于 2026-03-31 的设计文档整理，作为系统开发的阶段性计划。优先级基于"先跑起来，再优化"的原则。

---

## 一、技术栈确认

| 层级 | 技术选型 |
|---|---|
| 前端 | Vue 3 + MateChat + ECharts |
| 后端 | Python + FastAPI |
| 数据存储 | DuckDB + Parquet |
| AI 推理 | 本地 LLM（Ollama）或云端 API |
| 邮件推送 | SMTP |

---

## 二、开发阶段总览

```
阶段一：基础设施（核心骨架）
阶段二：数据层（原料供给）
阶段三：风控核心（防御侧跑通）
阶段四：知识库 + 机会嗅探（进攻侧跑通）
阶段五：报告系统（输出闭环）
阶段六：前端界面（用户交互）
阶段七：自动化 + 优化
```

---

## 三、阶段详解

### 阶段一：基础设施（核心骨架）

**目标**：搭建项目架子，前后端联调跑通，数据层就绪。

#### 1.1 项目初始化

- [ ] 创建 Git 仓库，初始化项目结构
- [ ] 前后端项目初始化（Vue 3 项目 + FastAPI 项目）
- [ ] 安装依赖：FastAPI / uvloop / DuckDB / Parquet 库 / Playwright 或 Scrapy
- [ ] 配置开发环境（Python 虚拟环境、Node 版本）

#### 1.2 数据层基础建设

- [ ] 设计 DuckDB 数据库 schema（建表 SQL 或 ORM 模型）
- [ ] 实现 Parquet 文件的读写工具函数
- [ ] 统一数据目录结构（data/user/、data/kv/、data/market/、data/rules/）
- [ ] 基础配置管理（config.yaml 或 .env）

#### 1.3 后端基础架构

- [ ] FastAPI 项目结构分层（routers / services / models / schemas）
- [ ] 用户数据模块（持仓、调仓、弱点画像的 CRUD）
- [ ] 规则模块（止盈规则、仓位规则的读写）
- [ ] 基础 API 验证（前后端联调）

#### 1.4 交付物
- 可运行的前后端骨架
- 数据层读写工具
- 基础 API 接口文档

---

### 阶段二：数据层（原料供给）

**目标**：各数据源接入系统，Parquet 文件能正常写入和查询。

#### 2.1 同花顺数据采集

- [ ] 同花顺页面结构分析（大盘情绪、板块资金流向、个股行情）
- [ ] 爬虫实现（Scrapy / Playwright）
- [ ] 数据清洗和格式化
- [ ] 写入 market/daily/ Parquet 文件
- [ ] 定时任务配置（收盘后自动抓取）

#### 2.2 雪球数据采集

- [ ] 雪球页面结构分析（大V发言列表、组合持仓）
- [ ] 登录态处理（Cookie / Token 维护）
- [ ] 爬虫实现
- [ ] 写入 kv/xueqiu_posts.parquet 和 kv/xueqiu_portfolios.parquet
- [ ] 轮询频率控制（避免封禁）

#### 2.3 公司公告采集

- [ ] 巨潮资讯页面结构分析
- [ ] 持仓股公告监控（按股票代码过滤）
- [ ] 写入 market/announcements.parquet
- [ ] 公告预警触发逻辑（利空分级）

#### 2.4 交付物
- 三个数据源可正常工作
- 每日数据自动写入 Parquet
- 告警机制就绪

---

### 阶段三：风控核心（防御侧跑通）

**目标**：风控逻辑完整运转，能生成合规评判和预警。

#### 3.1 止盈规则引擎

- [ ] 规则读取服务（四类止盈策略的参数化）
- [ ] 持仓扫描逻辑（逐票扫规则）
- [ ] 见顶信号判断（成交量/换手率暴增、高位区域、技术背离）
- [ ] 移动止盈线计算（20日/60日均线、最高点回撤）
- [ ] 即将触发预警计算

#### 3.2 仓位健康度检查

- [ ] 单票仓位计算
- [ ] 总仓位 / 现金比例计算
- [ ] 持仓数量检查
- [ ] 阈值配置化（可调整）

#### 3.3 用户弱点画像

- [ ] 历史交割单解析和上传
- [ ] 错误模式提取逻辑（场景聚类）
- [ ] 弱点清单生成
- [ ] 用户确认交互（弱点启用/禁用）
- [ ] 弱点动态更新流程

#### 3.4 交付物
- 止盈规则引擎完整
- 仓位检查可配置
- 弱点画像生成和更新流程跑通

---

### 阶段四：知识库 + 机会嗅探（进攻侧跑通）

**目标**：大V知识库正常入库，机会嗅探能给出"大V视角解读"。

#### 4.1 知识库管理

- [ ] 大V列表管理（用户添加/删除）
- [ ] 大V分级配置（核心/重点/辅助）
- [ ] 雪球发言的语义解析（提取标的、观点倾向、判断依据）
- [ ] 微信公众号文章录入（人工上传 + 解析）
- [ ] 分析师研报入库（雪球机构账号 or 专业平台）

#### 4.2 逻辑映射表

- [ ] 观点入库时的时间戳和标的记录
- [ ] 市场走势对照（定时任务，定期比对）
- [ ] 判断结果标注（成功/失败/无关）
- [ ] 大V权重动态更新

#### 4.3 机会嗅探模块

- [ ] 触发条件定义（大盘异动、持仓股异动、重大新闻）
- [ ] 知识库检索（相似情境查找）
- [ ] 多源观点聚合（按权重排序）
- [ ] 生成"大V视角解读"报告

#### 4.4 交付物
- 知识库可查询
- 大V权重动态调整
- 机会嗅探可输出

## 四、优先级建议

### 第一优先级（必做，先跑通闭环）
```
1. 阶段一（基础设施）
2. 阶段三（风控核心）— 防御侧先跑起来
3. 阶段五（报告系统）— 有输出才有反馈
4. 阶段二（数据层）— 燃料供给
```

### 第二优先级（进攻侧）
```
4. 阶段四（知识库 + 机会嗅探）
```

### 第三优先级（体验提升）
```
6. 阶段六（前端界面）
7. 阶段七（自动化）
```
