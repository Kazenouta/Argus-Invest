本风控系统主要包含三大模块：行为分析 + 规则约束 + 日报提醒，主要目的是基于用户历史教训和一系列风控规则，通过数据监控和规则强制，让用户管住手，防止非理性操作。

# 1. 核心设计理念

“风控的本质不是预测市场，而是约束人性。”

- 核心引擎：通过分析用户过去的交易记录，提取“常犯错误模式”（如：左侧接飞刀、盈利变亏损、追涨杀跌等）。
- 动态规则：基于提取的错误模式，自动生成针对性的“禁止操作清单”和“强制提醒规则”。
- 监控指标：每日关注市场行情和持仓情况，在盘前盘后发送两份简报，将“历史教训”与“当前风险”结合后推送给用户，一旦发现有指标异常（重点关注仓位管理和见顶信号），立即调用“历史教训”进行警告。

# 2. 系统架构

系统由三大模块组成，形成“分析->监控->提醒”的闭环

## 模块一：行为镜像引擎 —— 行为分析
- 功能：深度分析历史交割单，构建用户“弱点画像”。
- 输入：过去若干年的交易记录（Excel/CSV）。
- 处理逻辑：
  - 场景聚类：识别用户在什么情况下亏钱最多？例如“股价跌破60日线后补仓、高位爆量时追涨、单票仓位超过30%“。
  - 错误模式提取：生成《用户弱点清单》。例如“您在‘下跌趋势中补仓’的操作共15次，平均亏损12%，最大回撤35%。”
  - 规则映射：将弱点转化为具体的风控规则。
- 输出：动态规则库。

## 模块二：指标监控层 —— 规则约束
- 功能：实时扫描市场数据，提供触发风控规则的“信号”。
- 监控指标：
  - 仓位健康度：单票仓位是否 > 20%（常规上限）？总仓位是否 > 90%（现金<10%）？持仓数量是否 < 3 或 > 8？
  - 见顶风险信号：当日成交量/换手率 ≥ 4-5倍于均量？PE/PB分位点 > 80%？大盘高位 + 情绪亢奋？
- 作用：这些指标本身不直接给出买卖建议，而是作为触发器。一旦触发，立即唤醒“行为镜像引擎”中的对应规则，防止被市场氛围影响在股票高位盲目乐观。

## 模块三：双报提醒系统 —— 日报提醒
- 功能：在关键时间点推送结构化报告。
- 运行机制：
  - 晨报 (08:30 - 09:15)：预防性提醒。基于隔夜数据和用户弱点，列出今日“高危操作清单”和“禁止事项”。目的：让用户在开盘前冷静，预设心理防线。
  - 盘后复盘日报 (15:30 - 20:00)：反思性总结。对比今日实际操作与“弱点清单”，统计违规次数。展示若遵守规则可避免的损失（模拟推演）。目的：强化记忆，优化次日的“弱点画像”。

# 3. 核心工作流

## 3.1 初始化阶段
- 数据上传：用户上传历史交割单。
- 弱点提炼：系统自动生成《用户弱点清单》并让用户确认。确认项：“是否将‘下跌趋势补仓’列为最高级别禁止项？”
- 规则生成：系统根据确认的弱点，结合仓位关键和见顶信号，生成初始规则库。

## 3.2 日常运行流程
- 盘前扫描：检查持仓是否触发“见顶信号”（如某股票昨日爆量2倍以上）。检查仓位是否违反“结构律”（如单票被动超标）。生成《盘前风险提醒》，列出今日“禁止操作清单”。
- 盘后复盘：生成《合规性日报》。统计今日“违规次数”和“潜在避免损失”。若发现新的错误模式（如用户今天犯了一个以前没犯过的错），提示更新《用户弱点清单》。
